# AgentHarness 生命周期

`AgentHarness` 是低级代理循环之上的编排层。它拥有会话持久化、运行时配置、资源解析、操作锁定和面向扩展的变异语义。

本文档描述当前方向和已实现的行为。某些扩展/会话外观细节是计划中的，并明确标注。

## 最终生命周期目标

Harness 管理：
- 会话存储和持久化
- 模型和供应商配置
- 工具注册和活跃工具集
- 扩展加载和 hooks
- 上下文转换管道
- 操作锁定（防止并发提示）
- 转向和后续队列
- 压缩触发和执行
- 分支导航和摘要

## 操作模型

Harness 使用操作锁防止并发执行：

- `prompt()` 获取锁，运行代理循环，释放锁
- `compact()` 获取锁，运行压缩，释放锁
- `navigateTree()` 获取锁，导航分支，释放锁

排队的转向和后续消息在操作内部处理，不需要单独获取锁。

## 事件生命周期

完整的 prompt 操作：

1. 获取操作锁
2. 触发 `before_agent_start` hook（收集注入消息、系统提示修改）
3. 构建 LLM 上下文（transformContext + convertToLlm）
4. 调用供应商流
5. 处理助手消息和工具调用
6. 持久化到会话
7. 检查转向/后续队列
8. 释放锁

## 扩展集成

扩展通过 hooks 与 harness 交互：

- `context` - 在 LLM 调用前转换消息
- `before_provider_request` - 修改供应商请求
- `tool_call` - 预检/阻止工具调用
- `tool_result` - 后处理工具结果
- `message_end` - 观察完成的消息

扩展不直接访问 harness 内部。它们通过 hook 上下文中暴露的外观接口进行操作。
